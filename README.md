2.1 การ Normalize แต่ละ Factor (อธิบายวิธีและเหตุผล)

ใช้ Min-Max Normalization

สูตร:

x_norm=(x-x_min)/(x_max-x_min )

เนื่องจากแต่ละ factor มีหน่วยต่างกัน (HAND = เมตร, Slope = องศา, NDVI = -1 ถึง 1, TWI = ไม่มีหน่วย) ถ้าไม่ Normalize factor ที่มีค่าสูงกว่าจะครอบงำผลลัพธ์โดยอัตโนมัติ

และมีการ Invert (กลับค่า) บางปัจจัยเนื่องจากบางปัจจัย "ค่าสูง = เสี่ยงต่ำ" ต้องกลับเป็น 1 - normalized

ภารกิจที่ 4: Validation กับข้อมูลจริง

เปรียบเทียบผลลัพธ์โมเดลกับข้อมูลจริงที่หาได้ ตัวอย่างเช่น:

4.1 โมเดลน้ำท่วม → เปรียบเทียบกับ JRC Global Surface Water หรือ MODIS Flood

จากผลลัพธ์ในพื้นที่ที่เกิดน้ำน้ำท่วมชัดเจนตรงกับพื้นที่ที่ predict ไว้ในคลาสของ very high ส่วนบริเวณที่เป็นคลาสของ high ที่อยู่บริเวณใกล้เคียงกับคลาสของ very high มีบางพื้นที่ที่ไม่เกิดน้ำท่วมจริง ๆ 

4.2 ถ้าหาข้อมูล Validation ได้ยาก ให้อธิบายว่าทำไม และจะ Validate ได้อย่างไรในชีวิตจริง

หายากเพราะ 1. น้ำท่วมเป็นเหตุการณ์ชั่วคราว น้ำท่วมไม่ได้เกิดตลอดเวลา แต่โมเดลที่ทำเป็น risk (ความเสี่ยง) ไม่ใช่ event (เหตุการณ์จริง ณ เวลาใดเวลาหนึ่ง) ทำให้ match กันตรงๆยาก 

2. ข้อมูลดาวเทียมมีข้อจำกัด เช่น dataset อย่าง JRC Global Surface Water และเป็นข้อมูลสะสม (1984–2021) ไม่ได้แยก flood event รายปีชัดเจน และข้อมูลที่ได้แยกไม่ออกว่า น้ำถาวร เช่น แม่น้ำ/อ่างเก็บน้ำ หรือน้ำท่วมชั่วคราว ทำให้ validation อาจ overestimate น้ำท่วม

3. ปัญหา Cloud cover จำพวกข้อมูล optical เช่น Sentinel-2 มักมีเมฆช่วงน้ำท่วมทำให้ตรวจจับน้ำท่วมจริงได้ไม่ครบ 

4. ข้อมูลภาคสนาม (ground truth) หาได้ยากเนื่องจากไม่มี database flood extent ระดับจังหวัดแบบ open data ส่วนข้อมูลจากหน่วยงานรัฐอาจมีข้อมูลแต่ไม่เปิดเผย หรืออยู่ในรูป report / PDF 

วิธี Validate ในชีวิตจริง

1. ใช้ SAR data เช่น Sentinel-1 Radar มองทะลุเมฆได้ และตรวจน้ำท่วมช่วงเหตุการณ์จริงได้ วิธีการคือ เลือกช่วง flood event, ทำ water classification และเอามา compare กับ model 

2. ใช้ข้อมูลเหตุการณ์น้ำท่วมย้อนหลัง เช่น กรมป้องกันและบรรเทาสาธารณภัย (ปภ.), รายงานข่าว / disaster reports แล้วนำข้อมูลพื้นที่น้ำท่วมมาทำเป็น Flood points / flood polygons แล้วนำไป overlay กับ risk map
